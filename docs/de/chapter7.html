Kapitel 7: If-Anweisungen

Das letzte zu erlernende Steuerungskonstruktions-Steuerungskonstrukt für die Kernprogrammierung sind bedingte Anweisungen ( if und if/else).

Grundlegende Bedingungen

Eine if/else Anweisung führt genau dann einen "if" -Codeblock aus, wenn der bereitgestellte Test für den Zustand der Welt zum Zeitpunkt des Erreichens der Anweisung durch den Test wahr ist. Andernfalls führt das Programm den Codeblock "else" aus.

if(test){
wenn Codeblock
} else{
sonst Code-Block
}

Um ein Gefühl dafür zu bekommen, wo sich bedingte Anweisungen als nützlich erweisen könnten, schreiben wir ein Programm, in dem Karel eine Reihe von Piepsern invertiert. Wenn ein Feld zuvor einen Piepser hatte, sollte Karel es abholen. Wenn ein Feld keinen Piepser hat, sollte Karel einen niederlegen.

  • 1x1
  • 4x4
  • 3x6
  • 7x7
  • 8x8

Beachten Sie, dass eine if Anweisung keinen else Block enthalten muss. In diesem Fall funktioniert die Anweisung wie eine while Schleife, die nur einmal ausgeführt wird:

if(test){
wenn Codeblock
}

Bedingungen

Das letzte Beispiel verwendete eine neue Bedingung. Hier ist eine Liste aller Bedingungen, die Karel kennt:

Prüfung Gegenteil Was prüft es?
vorneIstKlar()frontIstBlockiert() Gibt es eine Wand vor Karel?
piepserVorhanden()keinePiepserVorhanden() Gibt es an dieser Ecke Piepser?
linksIstKlar()leftIstBlockiert() Gibt es eine Mauer zu Karels Linken?
richtigIstKlar()rechtIstBlockiert() Gibt es eine Wand zu Karels Recht?
piepserInDerTasche()keinePiepserInDerTasche() Gibt es Piepser in Karels Tasche?
nachNordenAusgerichtet()nichtNachNordenGerichtet() Ist Karel nach Norden ausgerichtet?
nachSüdenAusgerichtet()nichtNachSüdenAusgerichtet() Ist Karel nach Süden ausgerichtet?
nachOstenGerichtet()nichtNachOstenGerichtet() Ist Karel nach Osten gerichtet?
nachWestenGerichtet()nichtNachWestenGerichtet() Ist Karel nach Westen ausgerichtet?

Alles zusammenstellen

Glückwunsch! Sie kennen jetzt alle Kernsteuerungsblöcke für die Programmierung. Während Sie sie mit Karel, Methoden, Schleifen, für Schleifen gelernt haben, funktionieren die if / else -Anweisungen in fast allen Hauptsprachen, einschließlich Java, auf dieselbe Weise.

Nun, da Sie die Bausteine haben, können Sie sie zusammensetzen, um Lösungen für immer komplexere Probleme zu erstellen. Programmieren ist weitgehend die Wissenschaft der Problemlösung am Computer. Da Probleme häufig schwierig sind, können auch Lösungen - und die Programme, die diese Lösungen implementieren - schwierig sein. Um Ihnen die Entwicklung dieser Lösungen zu erleichtern, müssen Sie eine Methodik und eine Disziplin anwenden, die die Komplexität auf ein überschaubares Maß reduziert.
