Neue Befehle definieren

Glücklicherweise ermöglicht die Programmiersprache Karel die Definition neuer Befehle, indem einfach neue Methodendefinitionen eingefügt werden. Wenn Sie über eine Folge von Karel-Befehlen verfügen, die nützliche Aufgaben ausführen (z. B. rechts abbiegen), können Sie eine neue Methode definieren, die diese Befehlsfolge ausführt. Das Format für die Definition einer neuen Karel-Methode entspricht im Wesentlichen der Definition von run in den vorherigen Beispielen. Hierbei handelt es sich um eine eigene Methodendefinition. Eine typische Methodendefinition sieht folgendermaßen aus:

   private voidName() {
Befehle, die den Körper der Methode ausmachen
   }

In diesem Muster steht Name für den Namen, den Sie für die neue Methode gewählt haben. Um die Definition abzuschließen, müssen Sie lediglich die Reihenfolge der Befehle in den Zeilen zwischen den geschweiften Klammern angeben. Sie können biegenSieRechtsAb beispielsweise folgendermaßen definieren:

   private void biegenSieRechtsAb() {
biegenSieLinksAb();
biegenSieLinksAb();
biegenSieLinksAb();
   }

Ebenso können Sie eine neue drehDichUm Methode folgendermaßen definieren:

   private void drehDichUm() {
biegenSieLinksAb();
biegenSieLinksAb();
   }

Sie können den Namen einer neuen Methode wie alle in Karel integrierten Befehle verwenden. Wenn Sie biegenSieRechtsAb einmal definiert biegenSieRechtsAb, können Sie beispielsweise die drei Befehle biegenSieLinksAb im SteigKarel Programm durch einen einzigen Aufruf der Methode biegenSieRechtsAb. Es gibt natürlich einen offensichtlichen Unterschied zwischen den Definitionen der Methoden run und biegenSieRechtsAb im obigen Programm: Die Methode run ist im Gegensatz zu biegenSieRechtsAb, die als private gekennzeichnet ist, als public gekennzeichnet. Der Unterschied zwischen diesen beiden Bezeichnungen besteht darin, dass öffentliche Methoden von außerhalb des Programms aufgerufen werden können, private Methoden dagegen nicht. Die Run-Methode muss öffentlich sein, da die Karel-Umgebung in der Lage sein muss, einen Run-Befehl auszuführen, um die Dinge in Gang zu setzen. Im Allgemeinen ist es eine gute Programmierpraxis, Definitionen möglichst privat zu halten.

Methodencode-Blöcke

Eine Befehlsgruppe zwischen zwei geschweiften Klammern { } wird als Codeblock bezeichnet. Der Body Ihres Programms ist ein Codeblock, ebenso wie der Body einer Methode. Beachten Sie, wie der Inhalt eines Codeblocks alle um einen Tabulator eingerückt wird. Dies ist stilistisch wichtig, da der Benutzer leicht erkennen kann, welche Codezeilen sich in einem bestimmten Block befinden.

Sie können beliebig viele Methoden definieren. Sie sollten alle in Ihrem Programmcode-Block nacheinander geschrieben werden. Sie können keine Methode in einer anderen Methode definieren.
