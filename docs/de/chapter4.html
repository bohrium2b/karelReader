Kapitel 4: Zersetzung

Um mehr von der Kraft zu veranschaulichen, die mit der Definition neuer Methoden verbunden ist, ist es nützlich, dass Karel etwas praktischeres tut, als einen Piepser von einem Ort zum anderen zu bewegen. Die Straßen um Palo Alto scheinen oft reparaturbedürftig zu sein, und es macht Spaß zu sehen, ob Karel Schlaglöcher in seiner abstrakten Welt füllen kann. Stellen Sie sich zum Beispiel vor, dass Karel auf der „Straße“ steht, die in der linken Abbildung dargestellt ist, eine Ecke links von einem Schlagloch in der Straße. Karels Aufgabe ist es, das Loch mit einem Piepser zu füllen und zur nächsten Ecke überzugehen. Das Diagramm rechts zeigt, wie die Welt nach der Programmausführung aussehen soll.

Wenn Sie auf die vier vordefinierten Befehle beschränkt sind, würde die Ausführungsmethode zur Lösung dieses Problems folgendermaßen aussehen:

private void run() {
bewegung();
biegenSieLinksAb();
biegenSieLinksAb();
biegenSieLinksAb();
bewegung();
piepserSetzen();
biegenSieLinksAb();
biegenSieLinksAb();
bewegung();
biegenSieLinksAb();
biegenSieLinksAb();
biegenSieLinksAb();
bewegung();
}

Die anfängliche Motivation für die Definition der Methode biegenSieRechtsAb war, dass es mühsam war, drei biegenSieLinksAb-Befehle zu wiederholen, um eine Rechtskurve zu erreichen. Das Definieren neuer Methoden hat einen weiteren wichtigen Zweck, da Sie nicht jedes Mal die gleichen Befehlssequenzen wiederholen müssen, wenn Sie eine bestimmte Aufgabe ausführen möchten. Die Möglichkeit, Methoden zu definieren, macht die wichtigste Strategie beim Programmieren frei - das Aufteilen eines großen Problems in kleinere Teile, die leichter zu lösen sind. Das Aufteilen eines Programms in kleinere Teile wird als Zerlegung bezeichnet, und die Bestandteile eines großen Problems werden als Unterprobleme bezeichnet.

Als ein Beispiel kann das Problem des Füllens des Lochs in der Straße in die folgenden Teilprobleme zerlegt werden:

  1. Bewegen Sie sich zum Loch
  2. Füllen Sie das Loch, indem Sie einen Piepser hineinlassen
  3. Fahren Sie weiter zur nächsten Ecke

Wenn Sie auf diese Weise über das Problem nachdenken, können Sie mit Methodendefinitionen ein Programm erstellen, das Ihre Vorstellung von der Programmstruktur widerspiegelt. Die Run-Methode würde folgendermaßen aussehen:

private void run() {
bewegung();
schlaglochFüllen();
bewegung();
}

Die Übereinstimmung mit der Gliederung ist sofort klar, und alles wäre großartig, wenn Sie nur Karel dazu bringen könnten zu verstehen, was Sie unter schlaglochFüllen verstehen. Aufgrund der Möglichkeit, Methoden zu definieren, ist die Implementierung von schlaglochFüllen äußerst einfach. Alles, was Sie tun müssen, ist eine schlaglochFüllen-Methode zu definieren, deren Hauptteil aus den Befehlen besteht, die Sie bereits geschrieben haben, um den Job auszuführen:

private void schlaglochFüllen() {
biegenSieRechtsAb();
bewegung();
piepserSetzen();
drehDichUm();
bewegung();
biegenSieRechtsAb();
}

Hier ist das komplette Programm. Beachten Sie, wie Sie die Absicht des Programmierers durch das Lesen der run-Methode verstehen können.
