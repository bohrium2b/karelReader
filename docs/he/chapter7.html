פרק ז: אם הצהרות

הליבה הסופית של התכנות לבקרת זרימת הלימוד היא ללמוד אמירות מותנות ( if וכאשר if/else).

תנאי יסוד

הצהרה if/else מבצעת קוד "if" אם ורק אם הבדיקה המסופקת נכונה עבור מצב העולם בזמן התוכנית מגיע להצהרה. אחרת התוכנית מבצעת את "אחר" קוד הבלוק.

if(בדיקה){
אם לחסום קוד
} else{
מלבד קוד
}

כדי לקבל מושג איפה הצהרות מותנות עשוי להיות שימושי, בואו לכתוב תוכנית שיש לו קארל להפוך שורה של צפצופים. אם כיכר בעבר היה ביפר, קארל צריך להרים אותו. אם ריבוע אין ביפר, קארל צריך לשים אחד למטה.

  • 1x1
  • 4x4
  • 3x6
  • 7x7
  • 8x8

שים לב כי הצהרה if לא צריך להיות בלוק else- ובמקרה זה ההצהרה opperates כמו לולאה while זה רק פעם אחת:

if(בדיקה){
אם לחסום קוד
}

לשים את הכל ביחד

מזל טוב! עכשיו אתה יודע את כל הליבה שליטה בלוקים בלוקים. כאשר למדת אותם עם קארל, שיטות, בעוד לולאות, עבור לולאות, אם / אחר הצהרות לעבוד באותו אופן כמעט בכל השפות העיקריות, כולל ג 'אווה.

עכשיו שיש לך את אבני הבניין אתה יכול לשים אותם יחד כדי לבנות פתרונות לבעיות מורכבות יותר ויותר. במידה רבה, תכנות הוא מדע של פתרון בעיות על ידי המחשב. בגלל בעיות לעתים קרובות קשה, פתרונות - ואת התוכניות ליישם את הפתרונות הללו - יכול להיות קשה גם כן. על מנת להקל עליך לפתח פתרונות אלה, עליך לאמץ מתודולוגיה ומשמעת המפחיתה את רמת המורכבות הזו לסקאלה ניתנת לניהול.
